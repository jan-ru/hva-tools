# Requirements Document

## Introduction

A CLI tool written in Python that uses Playwright to extract rubric feedback from Brightspace DLO (a web-based Learning Management System). The tool connects to a manually authenticated browser session, navigates to specified class assignments, extracts rubric scores and feedback comments for student groups, and outputs aggregated feedback as one markdown file per group. The tool is designed to be extensible with additional CLI commands over time.

## Glossary

- **Extractor**: The CLI application that orchestrates browser automation, data extraction, and file output.
- **Brightspace**: The Brightspace DLO web-based Learning Management System where assignments and rubric feedback are hosted.
- **Browser_Session**: A Playwright browser instance where the user has already authenticated manually via SSO and 2FA.
- **Class**: A course or module within Brightspace that contains assignments and enrolled student groups.
- **Assignment**: A graded submission item within a Class that has an associated rubric with scores and feedback.
- **Rubric**: A grading structure attached to an Assignment, consisting of multiple criteria each with a score and optional feedback comment.
- **Criterion**: A single row in a Rubric, containing a name, score, and optional feedback comment.
- **Group**: A named collection of students who submit Assignments together within a Class.
- **Feedback_File**: A markdown file generated by the Extractor containing aggregated rubric feedback for a single Group across one or more Assignments.
- **Output_Directory**: The local filesystem directory where Feedback_Files are written.

## Requirements

### Requirement 1: Connect to Authenticated Browser Session

**User Story:** As a teacher, I want the tool to connect to a browser session where I have already logged in to Brightspace, so that I do not need to automate SSO and 2FA login flows.

#### Acceptance Criteria

1. WHEN the Extractor is launched, THE Extractor SHALL connect to an existing Playwright Browser_Session using the Playwright sync API.
2. WHEN the Extractor connects to the Browser_Session, THE Extractor SHALL verify that the user is authenticated in Brightspace by checking for the presence of a logged-in indicator on the page.
3. IF the Browser_Session is not authenticated, THEN THE Extractor SHALL display an error message indicating that manual login is required and exit with a non-zero exit code.

### Requirement 2: Accept Class and Assignment Configuration via CLI

**User Story:** As a teacher, I want to specify the class and list of assignments via CLI arguments, so that I can control which feedback to extract without editing configuration files.

#### Acceptance Criteria

1. THE Extractor SHALL accept a class identifier as a required CLI argument.
2. THE Extractor SHALL accept one or more assignment identifiers as required CLI arguments.
3. IF a required CLI argument is missing, THEN THE Extractor SHALL display a usage message describing the expected arguments and exit with a non-zero exit code.
4. THE Extractor SHALL use a CLI framework that supports adding subcommands, so that future commands can be added without restructuring the application.

### Requirement 3: Navigate to Class and Assignments in Brightspace

**User Story:** As a teacher, I want the tool to automatically navigate to the correct class and assignments in Brightspace, so that I do not have to manually browse to each page.

#### Acceptance Criteria

1. WHEN a valid class identifier is provided, THE Extractor SHALL navigate the Browser_Session to the corresponding Class page in Brightspace.
2. IF the specified Class is not found in Brightspace, THEN THE Extractor SHALL display an error message identifying the missing Class and exit with a non-zero exit code.
3. WHEN a valid assignment identifier is provided, THE Extractor SHALL navigate the Browser_Session to the corresponding Assignment submissions page within the Class.
4. IF a specified Assignment is not found within the Class, THEN THE Extractor SHALL display an error message identifying the missing Assignment and continue processing remaining Assignments.

### Requirement 4: Extract Rubric Feedback for All Groups per Assignment

**User Story:** As a teacher, I want the tool to extract rubric scores and feedback comments for every student group on each assignment, so that I have a complete record of the feedback I gave.

#### Acceptance Criteria

1. WHEN the Extractor navigates to an Assignment submissions page, THE Extractor SHALL identify all Groups that submitted the Assignment.
2. WHEN a Group submission is found, THE Extractor SHALL extract the names of all students belonging to the Group.
3. WHEN a Group submission is found, THE Extractor SHALL extract the Rubric associated with the submission, including each Criterion name, score, and feedback comment.
4. WHEN a Group submission is found, THE Extractor SHALL extract the date of the feedback or submission.
5. IF a Group submission has no Rubric feedback, THEN THE Extractor SHALL log a warning identifying the Group and Assignment and continue processing remaining Groups.

### Requirement 5: Aggregate Feedback Across Assignments per Group

**User Story:** As a teacher, I want feedback from multiple assignments to be collected into a single file per group, so that I can review all feedback for a group in one place.

#### Acceptance Criteria

1. WHEN feedback has been extracted from all specified Assignments, THE Extractor SHALL aggregate the extracted data by Group, combining feedback from all Assignments into a single data structure per Group.
2. THE Extractor SHALL preserve the association between each Rubric Criterion feedback entry and the Assignment it originated from.
3. THE Extractor SHALL order the aggregated feedback per Group chronologically by Assignment date.

### Requirement 6: Generate Markdown Feedback Files

**User Story:** As a teacher, I want the output saved as one markdown file per group, so that I can easily read and share the feedback.

#### Acceptance Criteria

1. WHEN aggregated feedback is ready for a Group, THE Extractor SHALL generate one Feedback_File in markdown format for that Group.
2. THE Feedback_File SHALL contain the Group name as a top-level heading.
3. THE Feedback_File SHALL contain the names of all students in the Group.
4. THE Feedback_File SHALL contain, for each Assignment, the Assignment name, date, and a table or list of Rubric Criterion names, scores, and feedback comments.
5. THE Extractor SHALL write all Feedback_Files to the Output_Directory.
6. IF the Output_Directory does not exist, THEN THE Extractor SHALL create the Output_Directory before writing files.
7. THE Extractor SHALL derive the Feedback_File filename from the Group name, using lowercase and replacing spaces with hyphens.

### Requirement 7: Provide Progress and Completion Feedback

**User Story:** As a teacher, I want to see progress while the tool runs, so that I know it is working and can estimate how long it will take.

#### Acceptance Criteria

1. WHILE the Extractor is processing Assignments, THE Extractor SHALL display the name of the Assignment currently being processed to the console.
2. WHILE the Extractor is processing Groups within an Assignment, THE Extractor SHALL display the name of the Group currently being processed to the console.
3. WHEN all processing is complete, THE Extractor SHALL display a summary indicating the number of Groups processed and the Output_Directory path.

### Requirement 8: Handle Navigation and Extraction Errors Gracefully

**User Story:** As a teacher, I want the tool to handle errors without crashing unexpectedly, so that I get as much feedback extracted as possible even if some pages fail.

#### Acceptance Criteria

1. IF a page navigation times out, THEN THE Extractor SHALL log an error message identifying the page and continue processing remaining items.
2. IF an expected page element is not found during extraction, THEN THE Extractor SHALL log a warning identifying the element and the Group or Assignment context and continue processing.
3. WHEN the Extractor encounters an unrecoverable error, THE Extractor SHALL display a descriptive error message and exit with a non-zero exit code.
